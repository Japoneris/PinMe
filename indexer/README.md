# Indexer

Scans whitelisted folders for images, computes a SHA-256 hash per file, and stores metadata in a SQLite database. Re-running is safe — already-indexed images are skipped.

## What gets stored (SQLite `images` table)

- `hash` — SHA-256 of file content (primary key, deduplication)
- `path` — absolute path on disk
- `mimetype`, `width`, `height`, `size_bytes` — file metadata
- `image_embedded`, `text_embedded` — flags consumed by the RAG pipeline
- `caption` — generated by the RAG pipeline (not populated here)

## Configuration

Edit `../config.json` to whitelist folders:

```json
{
    "explore": [
        "/home/user/Images/Projects",
        "/home/user/Photos"
    ]
}
```

## Run

```bash
pip install sqlalchemy pillow

python run.py                        # uses ../config.json and ../pintme.db
python run.py --config /path/to/config.json
python run.py --db /path/to/pintme.db
python run.py --quiet                # suppress per-file output
```

## Deduplication behaviour

| Case | Action |
|---|---|
| Same hash, same path | Skip |
| Same hash, new path | Update path (file was moved) |
| New path, different hash | Replace record, reset embedding flags |
| New file | Insert |
